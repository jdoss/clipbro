## Findings considered and rejected

Recorded so nobody re-audits them:

- **Per-keystroke filter memoization in the overlay** — the overlay loads at
  most 20 entries (`overlay.rs:189`); filtering 20 items per keystroke is
  microseconds. Not worth caching. (Plan 011 may change the entry count;
  re-evaluate only then.)
- **`wl_copy` zombie processes** — children are tokio `Child`s; tokio's
  orphan reaper collects them. No zombie accumulation. (The nice-to-have
  exit-status logging, plus a prompt reap of a self-exited *overlay* — an
  unrelated 11-min `<defunct>` seen on 2026-07-21 — are now plan 015.)
- **Selection-ownership "churn"** — one image copy produces a few Wayland
  selection-owner changes (source app → sync-to-primary → overlay-select).
  Normal for a syncing clipboard manager, and proven *not* to cause the
  2026-07-21 stale-paste report: the target app (Slack) ignores all external
  selection changes — a Chromium/Ozone-Wayland client cache cleared only by
  restarting Slack, not anything clipbro can influence by serving data
  differently. Reworking the serve path buys nothing here.
- **"Weak" SQLCipher key** — the 64-char alphanumeric key is ~381 bits of
  entropy through SQLCipher's KDF; fine. Plaintext-on-`encrypt_db=false` is
  explicit, documented user choice.
- **D-Bus `Store` size DoS** — any session peer can equally flood the
  clipboard itself via wl-copy; the D-Bus surface adds no new trust
  boundary on a session bus.
- **xdg-open scheme abuse (`file:`/`javascript:`)** — URL-typed entries
  only arise from the `^https?://\S+$` regex (`entry.rs:555-557`);
  `text/uri-list` entries have no `text_content()` so `OpenUrl` can't fire
  for them. By construction, only http(s) reaches xdg-open.
- **`db_path` path traversal** — the user configuring their own database
  path in their own config file is by design, not an attack.
- **"Daemon tests only test mocks"** — they assert on resulting database
  state; the few `withf` checks (sync direction/mime) are the behavior
  under test.
- **"entry.rs detection is untested"** — false; entry.rs has the repo's most
  thorough test module (detection, highlighting, URL classification).
- **`find_duplicate` two-queries-for-images micro-optimization, btn_style
  closure duplication, watcher-args duplication** — real but trivial;
  not worth standalone plans. Touch only if already editing those lines.
- **Unbounded daemon action channel** — events arrive at human clipboard
  rate; bounding it adds error paths for no demonstrated risk.
- **Pinning libcosmic to a `rev` in Cargo.toml** — lockfile + `--locked`
  install (plan 003) achieves reproducibility; a rev pin adds a second
  thing to bump on the maintainer's regular dep updates.
- **Private-IP/SSRF filtering for remote thumbnails** — opt-in feature, off
  by default; ureq's http(s)-only support is the de facto scheme allowlist
  (decode-bomb hardening is covered by plan 009).
- **rustfmt adoption** — would churn essentially every line; deferred as a
  maintainer style decision, deliberately excluded from CI (plan 003).
